# Introdução aos Sistemas Escaláveis

Componentizar vai além de fragmentar UI em pequenos pedaços. É pensar em como esses diferentes pedaços comunicam o estado, recebem propriedades e gerenciam as complexidades sem vazar lógica para fora do seu escopo.

## Single Responsibility Principle na UI

Ao aplicarmos SRP (Princípio da Responsabilidade Única) nos componentes de UI, tornamos nosso sistema imediatamente mais previsível. Um botão apenas exibe o visual e emite os cliques - quem gerencia *o que* deve ser feito no clique deve ser o container que o encapsula.

```tsx
function Button({ onClick, children }) {
  // Esse componente não faz requisições
  return <button onClick={onClick}>{children}</button>
}
```

Essa separação garante que possamos reutilizar o `<Button>` em centenas de locais sem reescrever a renderização de estados ou testes exaustivos de integração na camada visual.
